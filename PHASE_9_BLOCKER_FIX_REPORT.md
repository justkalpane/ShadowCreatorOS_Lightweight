## The Blocker

Packets generated by child workflows (WF-100, WF-200, WF-300, WF-500) were being linked to `DOSSIER-UNSPECIFIED` instead of real dossier IDs.

### Root Cause

1. **n8n webhook trigger wrapping**: When n8n's webhook trigger receives a POST request, it wraps the body under `$json.body`
2. **WF-010 Dossier Load Node mismatch**: The node was reading `$json.dossier_id` directly (top level), but it's actually at `$json.body.dossier_id`
3. **Fallback behavior**: When dossier_id is undefined, the node defaults to `DOSSIER-UNSPECIFIED`
4. **Cascade effect**: Downstream child workflows don't know which dossier to link packets to

### Evidence

Before fix:
```bash
$ curl http://localhost:5050/operator/outputs/DOSSIER-1777627000304-DN1Q7EMQ1
{
  "packets_count": 0,  ← No packets linked
  "packets": [],
  "grouped_by_type": {}
}

$ grep "DOSSIER-UNSPECIFIED" data/se_packet_index.json
21 occurrences ← Old packets still orphaned
```

## The Fix

**File Modified:** `/c/ShadowEmpire-Git/n8n/workflows/WF-010.json`  
**Node Updated:** "Dossier Load Node" (lines 22-32)

### Before (Broken)
```javascript
const dossier_id = $json.dossier_id || 'DOSSIER-UNSPECIFIED'
// Fails when dossier_id is nested in $json.body
```

### After (Fixed)
```javascript
const raw = $json || {};
const input = (raw.body && typeof raw.body === 'object') ? raw.body : raw;
const dossier_id = input.dossier_id || input.dossierId || input.dossier_context?.dossier_id || 'DOSSIER-UNSPECIFIED';
const route_id = input.route_id || input.routeId || 'ROUTE_PHASE1_STANDARD';
const user_mode = input.user_mode || input.userMode || 'operator';
const intent_id = input.intent_id || input.intentId || 'unknown';
const user_message = input.user_message || input.userMessage || '';

return [{ 
  json: { 
    ...input, 
    ...$json, 
    dossier_id,  // ← Now at top level for all downstream nodes
    route_id,
    user_mode,
    intent_id,
    user_message,
    dossier_context: { 
      dossier_id, 
      route_id, 
      loaded_at: new Date().toISOString() 
    } 
  } 
}];
```
