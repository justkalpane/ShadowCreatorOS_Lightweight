# MAC-06.0 Agentic Chat Runtime Model

## Purpose

Define the real lightweight Shadow Creator OS operating model across modern repo-aware agents before running reviewer proof phases.

## Lightweight Runtime Flow

1. User sends a content or strategy request.
2. Supported agent chat/coding agent receives the request.
3. Agent creates a Shadow Mission Packet.
4. Agent loads repo context through the Repo Context Loader.
5. Agent selects directors, agents, subagents, skills, and subskills from repo evidence.
6. Premium AI reasoning layer produces:
   - research brief
   - script v1
   - debate
   - critique
   - final script
   - final context packet
7. Quality gate checks topic adherence, grounding, JSON validity, lineage, and execution claims.
8. Output returns to chat and/or repo dossier.
9. n8n is used only when external execution is explicitly needed and approved.

## Supported Agent Layer

- Codex
- Claude
- ChatGPT
- Kimi
- DeepSeek
- Perplexity
- Gemini chat/app
- Antigravity
- Generic repo-aware coding agents

## Execution Boundary

Topic-to-context engineering does not require n8n, provider APIs, OpenWebUI, old Windows runtime, or media tooling.

## Primary Runtime Clarification

The primary model is universal single-agent repo-first operation. One capable repo-aware agent should be able to complete most intelligence work alone.

- Multi-agent review remains optional quality enhancement.
- Codex local repo-write operation is already proven through Dossier #2 and Dossier #3.
- Chat-only web/app agents can prove operation by returning complete Shadow output packets.
- Users should not need to install every supported agent before using the lightweight OS.
