# Repo Context Loader Protocol

Any supported agent must load the repo in this order before producing Shadow Creator OS outputs.

## Required Startup Read

1. `/Users/apple/Documents/ShadowCreatorOS_Lightweight/START_HERE_FOR_AGENTS.md`
2. `/Users/apple/Documents/ShadowCreatorOS_Lightweight/AGENT_READ_ORDER.md`
3. `/Users/apple/Documents/ShadowCreatorOS_Lightweight/AGENT_REPO_FIRST_OPERATING_DOCTRINE.md`
4. `/Users/apple/Documents/ShadowCreatorOS_Lightweight/AGENT_ANTI_DRIFT_RULES.md`

## Operating Standards

1. Read MAC-05 loop documents.
2. Read runtime contracts.
3. Load registries/directors/agents/skills only as needed for the mission.
4. Use historical docs only when deeper context is needed.
5. Never treat backup branches or old Windows runtime as active truth.

## Chat-Only Agent Mode

If the agent cannot write files, it must still produce a complete chat packet following `OUTPUT_TO_CHAT_CONTRACT.md`.

## Repo-Write Agent Mode

If the agent can write files, it may create dossier artifacts under the active repo contract after user approval.
