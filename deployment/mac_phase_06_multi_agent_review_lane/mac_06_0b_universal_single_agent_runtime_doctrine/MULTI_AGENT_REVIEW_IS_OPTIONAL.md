# Multi-Agent Review Is Optional

Multi-agent review is an optional quality enhancement, not a requirement for basic lightweight OS operation.

## What It Is Good For

- stronger quality review
- independent critique
- production readiness confidence
- specialized checks for logic, creativity, research, schema, or audience fit

## What It Is Not

- Not required for one capable agent to produce topic-to-context outputs.
- Not required for basic repo-first operation.
- Not a reason to force users to install every agent.

## Preservation Rule

Existing MAC-06 review lane docs remain valid and should not be deleted. They are future quality/review infrastructure.
