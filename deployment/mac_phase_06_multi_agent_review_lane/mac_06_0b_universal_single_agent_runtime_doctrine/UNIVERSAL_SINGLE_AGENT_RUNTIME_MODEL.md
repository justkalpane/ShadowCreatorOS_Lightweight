# Universal Single-Agent Runtime Model

## Core Doctrine

The primary lightweight Shadow Creator OS model is universal single-agent repo-first operation.

One capable repo-aware agent can run the intelligence layer by itself:

- Codex is one proven runtime, not the only required runtime.
- Claude, ChatGPT, Kimi, DeepSeek, Gemini, Perplexity, Antigravity, local LLM apps, and future repo-aware agents can operate if they can read the GitHub repo or receive repo context.
- The GitHub/local repo is the portable brain.
- The agent is the reasoning runtime.
- Dossiers and chat packets are production artifacts.

## What Single-Agent Operation Covers

Single-agent operation should cover roughly 80% of normal intelligence work:

- topic intake
- Shadow Mission Packet creation
- repo context loading
- registry-first director/agent/subagent/skill/subskill selection
- research
- script v1
- debate
- critique
- final script
- context engineering packet
- provider handoff packet
- quality gate
- lineage summary or lineage file

## What It Does Not Require

Basic lightweight OS operation does not require installing every agent. It does not require multi-agent review, n8n, provider APIs, OpenWebUI, or media tooling.

## Future Execution Layer

n8n is only for external execution later: voice, avatar, image/video, publishing, analytics, provider orchestration, and artifact/status callbacks.
