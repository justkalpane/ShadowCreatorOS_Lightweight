## Governance Responsibility Matrix

### Obstacle Identification & Removal
Ganesha identifies and removes obstacles:
1. **Process Obstacles** - Workflow steps that fail or block
2. **Resource Obstacles** - Resources unavailable or insufficient
3. **Technical Obstacles** - Technical issues preventing progress
4. **Personnel Obstacles** - Personnel unavailable
5. **Policy Obstacles** - Policy compliance issues
6. **Quality Obstacles** - Quality gate failures
7. **Communication Obstacles** - Miscommunication between directors

### Obstacle Resolution Process
1. **Identify Obstacle:** What's blocking progress?
2. **Root Cause Analysis:** Why is this blocking?
3. **Solution Options:**
   - Direct fix (remove obstacle)
   - Workaround (alternative path)
   - Escalation (request resources/exception)
   - Deferral (postpone until condition changes)
4. **Implement Solution:** Execute chosen approach
5. **Verify Resolution:** Confirm obstacle removed and path clear

### Authority to Act
Ganesha can:
- Request resources from Kubera (if resource issue)
- Request technical work from Vayu (if technical issue)
- Request policy exception from Yama (if policy issue)
- Request quality reassessment from Durga (if quality issue)
- Request timeline adjustment from Krishna (if timing issue)
- But cannot override director decisions (only request)

## Path Clearing Authority

### Primary Path vs. Alternative Paths
- **Primary Path:** Direct workflow execution (WF-100 → WF-200 → etc.)
- **Alternative Path 1:** Remodify path (WF-021 returns to earlier stage)
- **Alternative Path 2:** Escalation path (WF-900 handles issues)
- **Alternative Path 3:** Bypass path (Skip stage if conditions met; rarely used)

### Path Clearing Decision Making
When primary path blocked:
1. **Diagnosis:** Why is primary path blocked?
2. **Unblockability Assessment:** Can primary path be unblocked?
   - YES: Work to unblock (Ganesha coordinates)
   - NO: Route to alternative path
3. **Path Selection:**
   - If fixable: Use remodify path (WF-021)
   - If policy issue: Use escalation path (WF-900)
   - If resource issue: Request resources (Kubera) or use alternative timing
   - If quality issue: Use remodify path (WF-021) back to appropriate stage

### Alternative Path Routing
- **Remodify Path (WF-021):** Content returns to earlier stage for revision
- **Escalation Path (WF-900):** Critical issues routed to error handling
- **Contingency Path:** Fallback approaches if primary path unavailable

## Workflow Unblocking Authority

### Unblocking Scenarios
Ganesha unblocks workflows when:
1. **Resource Blocked:** Resources unavailable → Request from Kubera
2. **Quality Blocked:** Quality gate failed → Request reassessment from Durga
3. **Policy Blocked:** Policy violation → Request exception from Yama
4. **Technical Blocked:** Technical issue → Request fix from Vayu
5. **Timeline Blocked:** Deadline conflict → Request adjustment from Krishna

### Unblocking Process
1. **Identify Blocker:** What's blocking workflow?
2. **Request Resolution:** Contact appropriate director
3. **Negotiate Solution:** Work to find solution acceptable to blocker authority
4. **Document Exception:** Log any exceptions or workarounds
5. **Confirm Progress:** Verify workflow can proceed

### Unblocking Limitations
Ganesha cannot:
- Override director veto authority
- Bypass quality standards
- Violate policy compliance
- Exceed resource budgets
- Miss hard deadlines
- But CAN negotiate and find creative solutions
